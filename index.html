Abstract

Comparing and computing distances between phylogenetic trees are important biological problems, especially for models where edge lengths play an important role. A space modeling the set of all phylogenetic trees having a fixed set of leaves is considered. This space provides a way of measuring distance between phylogenetic trees. It also provides a justification for disregarding portions of a collection of trees that agree, thereby, simplifying the space in which comparisons are to be made. The geodesic distance measure between two phylogenetic trees with edge lengths is the length of the shortest path between them in this continuous tree space. We aim to understand and implement different algorithms that provide a way to compute geodesic distances in tree space. Specifically, we would be looking at three different algorithms including the GTP algorithm, GeodeMaps and GeoMe Tree.

Plan of Action

The motivation behind this project is to compute the shortest path between phylogenetic trees in Euclidean tree space. We plan to do a comparative study of different algorithms which help calculate this shortest distance after which the results are recorded to give us an idea about the efficiency of each one of them

Algorithms

1. GTP
2. GeodeMap
3. GeotMeTree

Dataset Used

The dataset being used for the implementation is TreeBASE. It is a repository containing all phylogenetic information, phylogenetic trees and the data needed to generate them.

Objective

Java and Python implementation followed by a time comparison of the result of each algorithm.

Work Distribution

Since three algorithms are being tested, the workload will be equally split among all the project members. The tasks that will be undertaken for each algorithm include
1. Understanding the working of the algorithm
2. Testing and evaluation of data during algorithmic implementation.
3. Comparing the results of one algorithm with the result of others.
